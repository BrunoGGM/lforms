    • LForms Demos

    A widget for rendering feature-rich forms defined in JSON definition files

    The following demos introduce LForms a few features at a time using simple forms. For a set of demos showing of the full feature set of LForms, including skip logic and score rules, see the lforms-demo website, or see the last demo, which lets you pull in LOINC forms.

    1. A form with just a single question. The form definition (which you can see on the "JavaScript" tab of the demo) only defines the question "Eye color". By default, the form template includes some top level questions seen here (Date Done, etc.) Also, the form template includes some checkboxes. If you click on "Display Question Code" you will see the code for both the form and the question. Both of these these parts are configurable and removable.
      [Show Demo]
    2. This is the same form but configured to turn off both the checkboxes and the top-level questions. (In the form definition, obrHeader is what controls the visibility of the top-level questions.)
      [Show Demo]
    3. If you do want the top-level questions, but want different ones, that can be controlled as well. Here is an example with different fields.
      [Show Demo]
    4. Questions can can coded answer lists. The dataType for list questions should be set to either "CNE" ("coded with no exceptions", which limits input to list choices) or "CWE" ("coded with exceptions", which allows entry of off-list answers).
      [Show Demo]
    5. Questions can also be date fields with a calendar input, or be numeric fields with units.
      [Show Demo]
    6. When answer lists are too large to be included directly in the form definition, they can be specified via a URL which returns matches as the user types. Several such data sources are available on lforms-service. (If you wish to construct your own, the web service receiving the URL will need to understand certain parameters and return data in the expected format. For information on how to do that, see the documentation for the autocomplete-lhc search list, a look at the requirements for the "url" parameter to the constructor.)
      [Show Demo]
    7. TBD -- More demos to come, showing multi-answer questions, sections, repeating sections, questions in tables, skip logic, and more!
    8. The RxTerms demo form. This simple form just has two fields, one which provides a search autocompleter and one which obtains its list of strengths from the data associated with the selected drug name.
      [Show Demo]
    9. LOINC forms. The autocompleting field at the top lets you select a form which is then rendered below.
